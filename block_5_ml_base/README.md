## Задание 1: Пони тоже кони

Для дальнейшей работы следует выбрать модель с **ROC-AUC = 0.7**.

Обоснование:
- Значение ROC-AUC = 0.7 выше случайного угадывания (0.5) и говорит о наличии полезного сигнала в данных.
- Модель с ROC-AUC = 0.1 работает хуже случайной и, вероятно, либо обучена с ошибкой, либо предсказывает инвертированные классы.

Дальнейшие действия с выбранной моделью:
- Проверить корректность разметки классов.
- Выполнить тюнинг гиперпараметров.
- Провести кросс-валидацию.
- Попробовать улучшить качество за счёт feature engineering.
- Проверить модель на отложенной выборке.

## Задание 2: Ручной счёт ROC-AUC

ROC-AUC показывает вероятность того, что случайно выбранный положительный объект получит более высокий скор, чем случайно выбранный отрицательный.

Для ручного расчёта:
- Берутся все возможные пары (positive, negative).
- Подсчитывается доля пар, в которых score(positive) > score(negative).
- При равенстве значений добавляется 0.5.

Формула:
ROC-AUC = (количество корректных сравнений + 0.5 × количество равенств) / общее количество пар.

## Задание 3: Ручной счёт корреляции

По рассчитанному коэффициенту корреляции Пирсона наблюдается **отрицательная линейная зависимость** между количеством выпитых чашек кофе и итоговым баллом за экзамен.

Выводы:
- С увеличением количества кофе в среднем балл за экзамен снижается.
- Наличие корреляции **не означает причинно-следственную связь**.
- Нельзя утверждать, что кофе напрямую ухудшает результаты экзамена, так как возможны скрытые факторы (стресс, недосып, сложность экзамена и т.д.).
